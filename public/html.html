Introduction

Before starting with the interview questions on HTML Language, let’s first go through HTML, what is HTML, the career opportunities it provides, etc. HTML stands for HyperText Markup language. It is a standard text formatting language used for developing web pages released in 1993. HTML is a language that is interpreted by the browser and it tells the browser what to display and how to display. HTML is an important language to learn if anyone wants to work in the web development domain [Web designers, Web Developers]. HTML alone is not sufficient for a web developer because HTML only defines the structure of the data that will be rendered on the browser in a webpage, to make it visually appealing and to make it functional, we will need to use CSS and Javascript respectively.

1. Are the HTML tags and elements the same thing?

No,HTML elements are defined by a starting tag, may contain some content, and a closing tag. For example, <h1>Heading 1</h1> is an HTML element, but just <h1> is a starting tag and </h1> is a closing tag.

In the first example, <h1>Heading 1</h1>, the content "Heading 1" will be displayed as a top-level heading (h1) on the web page because it is enclosed between the opening and closing tags of the h1 element.

However, in the second example, <h1>, there is no content between the opening and closing tags. This is just a starting tag, and without the closing tag, it doesn't form a complete HTML element.

Similarly, </h1> is only a closing tag. It indicates the end of an h1 element, but it doesn't create a complete HTML element on its own.

2. What are tags and attributes in HTML?

Tags are the primary component of HTML that defines how the content will be structured/formatted, whereas Attributes are used along with the HTML tags to define the characteristics of the element.

For example, <p align="center">Interview questions</p>, in this example, the ‘align’ is the attribute using which we will align the paragraph to show in the center of the view.

The HTML tag <p> represents a paragraph element, and the ‘align’ attribute is used with the <p> tag to specify that the paragraph content should be aligned to the center of the view.

Attributes can be used with various HTML tags to add functionalities or styles to the elements. They play a crucial role in controlling the behavior and presentation of the content on a web page. It's important to use attributes appropriately to enhance the overall user experience and accessibility.

3. What are void elements in HTML?

HTML elements that do not have closing tags or do not need to be closed are known as Void elements or Self-closing elements. Examples of void elements in HTML include <br />, <img />, <hr />, and more.

Void elements are used to insert content into a web page that doesn't require additional content or nested elements. These elements are self-contained and don't have any content between an opening and closing tag.

For example, the <br /> tag represents a line break, <img /> is used to embed images, and <hr /> represents a horizontal rule (a horizontal line) to separate content. None of these void elements require a separate closing tag.

When using void elements, it's essential to remember to include the trailing slash ("/") before the closing angle bracket to ensure they are interpreted correctly by all browsers.

4. What is the advantage of collapsing white space?

HTML, a blank sequence of whitespace characters is treated as a single space character, Because the browser collapses multiple spaces into a single space character and this helps a developer to indent lines of text without worrying about multiple spaces and maintain readability and understandability of HTML codes

5. What are HTML Entities?

In HTML, some characters are reserved, such as <, >, /, etc. To use these characters in our webpage, we need to use character entities called HTML Entities. Below are a few mappings between the reserved characters and their respective entity characters to be used.

Character Entity Name Entity Number
< &lt; <
> &gt; >
& &amp; &
(non-breaking space) Eg. 10 PM &nbsp; Eg. <p>10&nbsp;&nbsp;PM</p>  

For example, to display the less-than sign (<) on a webpage, we use &lt; or < as an HTML entity. Similarly, to display the greater-than sign (>), we use &gt; or > as an entity. And to display an ampersand (&), we use &amp; or & as an entity.

Additionally, the non-breaking space ( ) is represented by &nbsp; or   as an entity. This ensures that multiple consecutive spaces are not collapsed into a single space by the browser.

10. Describe HTML layout structure.

Every web page has different components to display the intended content and a specific UI. However, there are some templated elements that are globally accepted as a standard way to structure a web page. Some of these commonly used components include:

  • <header>: Stores the starting information about the web page.
  • <footer>: Represents the last section of the page.
  • <nav>: The navigation menu of the HTML page.
  • <article>: Represents a self-contained set of information that could be independently distributed or reused.
  • <section>: Used to define the basic structure of a page and can be used within the <article> block.
  • <aside>: Represents the sidebar content of the page.

These elements help provide a more semantic and organized structure to the web page, making it easier for search engines and screen readers to understand the content and improve overall accessibility.

11. How to optimize website assets loading?

To optimize website load time, various techniques can be employed to improve asset loading. Some of these optimization methods include:

  • CDN Hosting: A CDN or content delivery network is a network of geographically distributed servers that help reduce latency by delivering assets from the server closest to the user's location.
  • File Compression: This method reduces the size of assets (e.g., images, scripts, stylesheets) to minimize data transfer, resulting in faster loading times.
  • File Concatenation: Combining multiple files into a single file reduces the number of HTTP calls required to fetch assets, reducing loading time.
  • Minify Scripts: Minification removes unnecessary characters from scripts and CSS files, reducing their overall file size and speeding up loading times.
  • Parallel Downloads: Hosting assets on multiple subdomains can help bypass the download limit of 6 assets per domain of modern browsers, improving overall loading performance.
  • Lazy Loading: Instead of loading all assets at once, non-critical assets can be loaded on an as-needed basis. This technique improves initial page load times and reduces the load on the server.

Implementing these optimization techniques can significantly enhance the user experience by reducing page load times and improving website performance.

12. What are the various formatting tags in HTML?

HTML has various formatting tags that can be used to style the text content. Some of these formatting tags include:

  • <b>: Makes the text bold.
  • <i>: Makes the text italic.
  • <em>: Makes the text italic but with added semantic importance.
  • <big>: Increases the font size of the text by one unit.
  • <small>: Decreases the font size of the text by one unit.
  • <sub>: Makes the text a subscript.
  • <sup>: Makes the text a superscript.
  • <del>: Displays the text as strikeout text.
  • <strong>: Marks the text as important with added semantic importance.
  • <mark>: Highlights the text with a background color.
  • <ins>: Displays the text as added text with underline.

These formatting tags allow you to add visual emphasis and semantic meaning to different parts of your text content in HTML.

13. What are the different kinds of Doctypes available?

In HTML, there are three kinds of Doctypes (Document Type Declarations) available:

  1. Strict Doctype: It is used to enforce strict HTML standards, and it does not allow the use of deprecated elements or attributes. It is defined as follows: 
    <!DOCTYPE html>
  2. Transitional Doctype: It allows the use of deprecated elements and attributes, which were used in older versions of HTML. It is defined as follows: 
    <!DOCTYPE html PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd">
  3. Frameset Doctype: It is used when the web page contains frames. Frames are an older and now less commonly used feature of HTML. It is defined as follows: 
    <!DOCTYPE html PUBLIC "-//W3C//DTD HTML 4.01 Frameset//EN" "http://www.w3.org/TR/html4/frameset.dtd">

It is recommended to use the Strict Doctype (<!DOCTYPE html>) for modern web development as it enforces adherence to the latest HTML standards and encourages the use of more modern and cleaner code.

15. What is the difference between < strong >, <b> tags and <em>, <i> tags?

The effect on a normal webpage of the tags <strong>, <b>, <em>, and <i> is often similar in terms of font styling. Both <b> and <i> tags are used to apply font styling, with <b> representing bold and <i> representing italic text. However, these tags don't provide any semantic meaning to the text.

The key difference lies in the <strong> and <em> tags. Both tags have a semantic meaning to emphasize certain parts of the text:

  • <strong>: The <strong> tag represents text of strong importance or greater significance. It is used to emphasize the importance of the enclosed text. Visually, it may appear bold, but its primary purpose is to add semantic weight to the content.
  • <em>: The <em> tag represents text with emphasis. It indicates that the enclosed text should be emphasized or displayed in an italicized style. Like <strong>, its primary function is to provide semantic importance to the content.

It is recommended to use <strong> and <em> tags when the emphasis and importance of the content matter, as they not only provide styling but also convey meaningful information to assistive technologies and search engines.
